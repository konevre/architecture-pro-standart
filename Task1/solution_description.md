# Решение Задачи 1: Карта IT-ландшафта и Схема Интеграции

## 1. Карта IT-ландшафта

Карта IT-ландшафта (сущности и связи) уже сформирована в файле `it_landscape.drawio`. 
Текущее распределение систем по функциональным областям:

| Подразделение (Строки) | Продажи / Обслуживание (Колонки) | Используемые системы |
|------------------------|----------------------------------|----------------------|
| **Сеть отделений**     | Продажи продуктов                | АБС (Десктоп клиент) |
| **Кол-центр**          | Поддержка / Продажи              | Система КЦ (Vendor)  |
| **Бэк-офис (Депозиты)**| Расчет ставок                    | Excel, Почта, АБС    |
| **Бэк-офис (Кредиты)** | Андеррайтинг                     | Кредитный конвейер (Camunda) |
| **IT / Риски**         | Скоринг                          | Система Скоринга (Python) |
| **Клиент**             | Самообслуживание                 | Интернет-банк, Сайт  |

## 2. Описание Кредитного Процесса (Current State)

Процесс выдачи кредита в текущей архитектуре характеризуется низкой скоростью из-за суточной синхронизации данных.

### Участники процесса:
1.  **Клиент**
2.  **Менеджер отделения (Фронт-офис)**
3.  **АБС** (Мастер-система для заявок)
4.  **Кредитный конвейер** (Camunda)
5.  **Система Скоринга**
6.  **БКИ** (Бюро кредитных историй)

### Алгоритм процесса:

1.  **Подача заявки:**
    *   Клиент обращается в отделение.
    *   Менеджер заводит заявку в **АБС**.
    *   *Проблема:* Заявка "лежит" в АБС до ночной выгрузки.

2.  **Передача в обработку:**
    *   Раз в сутки (Batch job) данные из АБС передаются в **Кредитный конвейер** (интеграция через БД).

3.  **Андеррайтинг и Скоринг:**
    *   Сотрудники кредитного отдела работают в **Кредитном конвейере**.
    *   Конвейер вызывает **Систему Скоринга** через REST API.
    *   **Система Скоринга**:
        *   Запрашивает данные в **БКИ** (онлайн).
        *   Использует данные из **АБС** (история платежей), которые обновляются в её локальной базе раз в сутки.
        *   Возвращает итоговый балл в Конвейер.

4.  **Принятие решения:**
    *   На основании скоринга сотрудник принимает решение в Кредитном конвейере.

5.  **Финализация:**
    *   Финальный статус заявки передается обратно в **АБС** (раз в сутки).
    *   После получения одобренного статуса в АБС происходит зачисление средств (выдача кредита).

**Ключевое ограничение:** Минимальный срок выдачи кредита — 2 дня (из-за двойной суточной синхронизации АБС <-> Конвейер).

## 3. Схема Интеграции Приложений

Схема интеграции подготовлена в формате PlantUML в файле `integration_schema.puml`. Она отображает:
*   Прямую связь Интернет-банка и БД АБС.
*   "Ручную" интеграцию через Email для депозитов.
*   Асинхронный обмен данными между АБС и Кредитным конвейером.
